**Timeline:** 4/5 → 10/5 (5 working days)

## Day 1 - 4/5: Setting Up AWS Billing Alerts

**Tasks Completed:** I studied AWS Billing and Cost Management to understand how to monitor and control cloud expenses. I then configured Billing Alerts using Amazon CloudWatch to track account spending and receive notifications whenever the estimated cost exceeded predefined thresholds.

**Knowledge Gained:** I learned the importance of monitoring cloud costs when using AWS services. Billing Alerts help users proactively manage their budgets and prevent unexpected expenses during system development and learning activities.

**Achievements:** I successfully configured AWS Billing Alerts and understood how AWS monitors account spending in real time.

**Challenges and Lessons Learned:** Initially, I did not fully understand the relationship between AWS Billing, Amazon CloudWatch, and Amazon SNS. After studying the documentation and practicing several times, I gained a clear understanding of how these services work together to deliver cost notifications.

## Day 2 - 5/5: AWS Budgets

**Tasks Completed:** I continued learning about AWS Budgets and practiced creating monthly budget plans. I configured multiple budget thresholds to receive notifications before the account exceeded the predefined spending limits.

**Knowledge Gained:** I learned that AWS Budgets can monitor not only overall costs but also service usage, enabling organizations to manage cloud spending more effectively throughout the project lifecycle.

**Achievements:** I successfully created AWS Budgets for my account and configured budget alerts suitable for the internship environment.

**Challenges and Lessons Learned:** AWS cost data is updated periodically rather than instantly, so budget alerts may not appear immediately. I learned that cost monitoring requires continuous observation over time to obtain accurate results.

## Day 3 - 6/5: Designing a Role-Based Access Control (RBAC) Model

**Tasks Completed:** I studied the Role-Based Access Control (RBAC) model and applied it to our internship project. Together with my teammates, I identified the primary user roles, including Admin, Security Manager, Security Analyst, and Auditor, and defined the permissions assigned to each role within the system.

**Knowledge Gained:** I learned that RBAC simplifies permission management by assigning access rights based on roles instead of individual users. This approach improves system security, simplifies administration, and reduces the likelihood of permission-related errors.

**Achievements:** Our team finalized the primary system roles and completed the permission specification for each role in preparation for implementation during the following development phases.

**Challenges and Lessons Learned:** Determining the appropriate permissions for each role required careful analysis to balance system security with practical operational requirements.

## Day 4 - 7/5: Forming the Project Team and Planning

**Tasks Completed:** I participated in a team meeting to define the objectives of our internship project. We discussed the project scope, selected the technologies to be used, and established a development roadmap for each stage of the implementation process.

**Knowledge Gained:** I gained a better understanding of the software development workflow, including requirement analysis, task planning, and project scheduling. I also realized that regular communication is essential for effective teamwork.

**Achievements:** The team successfully agreed on the project's overall direction and established a detailed development plan for the upcoming weeks.

**Challenges and Lessons Learned:** At first, team members had different opinions regarding the system implementation approach. Through discussion and evaluation, we reached a solution that best satisfied the project requirements.

## Day 5 - 8/5: Task Assignment

**Tasks Completed:** After completing the overall project plan, the team assigned responsibilities to each member. I was responsible for researching AWS services related to backend development and cloud security while preparing the technical documentation required for the implementation phase.

**Knowledge Gained:** I learned that clearly assigning responsibilities helps prevent overlapping work and improves team productivity. I also realized the importance of regularly updating progress to ensure the project stays on schedule.

**Achievements:** The team successfully distributed tasks, clearly defined each member's responsibilities, and prepared the necessary documentation for the upcoming development stage.

**Challenges and Lessons Learned:** I realized that communication and collaboration skills are just as important as programming skills in a software development project. Regular progress updates help identify issues early and minimize their impact on the overall project.

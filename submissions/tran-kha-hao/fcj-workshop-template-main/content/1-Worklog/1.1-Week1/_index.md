**Timeline:** 20/4 → 24/4 (5 working days)

## Day 1 - 20/04: Setting Up an AWS Account

**Tasks Completed:** I spent time learning about Amazon Web Services (AWS), cloud computing concepts, and its core services. I also read the company's onboarding materials and explored the AWS Management Console to understand its interface, service categories, and resource management.

**Knowledge Gained:** I learned that AWS is a cloud computing platform providing infrastructure through IaaS, PaaS, and SaaS models. I also became familiar with the AWS Console and its main service categories, including Compute, Storage, Database, Networking, and Security.

**Achievements:** I gained an overall understanding of the AWS ecosystem and learned how to navigate the AWS Management Console, providing a solid foundation for the upcoming labs.

**Challenges and Lessons Learned:** At first, the large number of AWS services was overwhelming and difficult to remember. I learned that grouping services by their functions is much more effective than trying to memorize them individually.

## Day 2 - 21/4: Studying the Workshop Template

**Tasks Completed:** I reviewed the internship Workshop Template to understand its structure, documentation style, lab procedures, and the overall learning objectives throughout the internship program.

**Knowledge Gained:** I gained a clear understanding of the AWS internship learning roadmap, documentation organization, and the workflow for completing each workshop.

**Achievements:** I established a study plan for each stage of the internship and understood the objectives of every workshop.

**Challenges and Lessons Learned:** Since the documentation was quite extensive, I realized the importance of taking notes on key sections for easier reference later.

## Day 3 - 22/4: Registering an AWS Account

**Tasks Completed:** I registered a new AWS account, completed email and phone verification, and configured the payment method according to the company's instructions.

**Knowledge Gained:** I learned the AWS account registration process and the required verification steps to activate all available services.

**Achievements:** My AWS account was successfully created and activated.

**Challenges and Lessons Learned:** I learned that verifying account information carefully is essential to avoid registration errors and activation delays.

## Day 4 - 23/4: Configuring the AWS Environment

**Tasks Completed:** I configured the default AWS Region, explored the Billing Dashboard and Cost Management features, and reviewed the basic account settings required for future labs.

**Knowledge Gained:** I learned the purpose of AWS Regions and Availability Zones, as well as how to select the appropriate Region based on project requirements.

**Achievements:** I completed the initial AWS environment configuration and prepared the account for upcoming hands-on exercises.

**Challenges and Lessons Learned:** I realized that using a consistent Region throughout the project is important to avoid resource management issues and deployment errors.

## Day 5 - 24/4: Finalizing the Environment Setup

**Tasks Completed:** I reviewed the AWS account configuration, verified access permissions, studied AWS security best practices, and prepared the environment for the following week's training.

**Knowledge Gained:** I learned the importance of securing the Root Account, enabling Multi-Factor Authentication (MFA), and configuring Billing Alerts to monitor AWS costs.

**Achievements:** The AWS environment was fully prepared and ready for the upcoming hands-on labs.

**Challenges and Lessons Learned:** I learned that the Root Account should only be used for critical administrative tasks, while daily operations should be performed using IAM Users to enhance security.
